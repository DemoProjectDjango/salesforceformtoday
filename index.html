1. App Purpose

Salesforce Form Today is designed to help organizations efficiently transfer data collected through Google Forms into Salesforce without manual data entry.

Many organizations use Google Forms to collect structured information such as registrations, surveys, feedback, applications, or internal requests. Salesforce Form Today provides a secure and controlled way to synchronize this data into Salesforce where it can be managed, reported on, and acted upon.

Primary Purpose

The primary purpose of the application is to enable Salesforce users to:

  • Connect a Google account through OAuth 2.0 authentication
  • Select Google Forms that they own or manage
  • Read the structure and questions of those forms
  • Retrieve form responses submitted by respondents
  • Insert or update corresponding records inside Salesforce based on user-defined mappings

The application operates entirely under user control. All Google data access is explicitly initiated by the user from within Salesforce and occurs only after the user grants consent.

2. Purpose of Google API Access

Salesforce Form Today requires access to Google APIs solely to support its core synchronization functionality.

Google APIs are used only to:

  • Identify Google Forms owned by the authenticated user
  • Read form metadata such as form name, questions, and field types
  • Retrieve responses submitted to selected forms
Note: No Google APIs are used for unrelated purposes.

The application does not request access to Gmail, Google Contacts, Google Calendar, Google Photos, or any other Google services outside of Google Forms and limited Google Drive access required to locate forms.

3. What This App Does

Salesforce Form Today enables Salesforce users to:

  • Connect a Google account using OAuth 2.0 authentication
  • View and select Google Forms owned by the authenticated user
  • Read Google Form structure and questions
  • Map Google Form questions to Salesforce object fields
  • Synchronize Google Form responses into Salesforce records
  • Manually control when synchronization runs from Salesforce

All actions are initiated by the user from inside Salesforce.

4. Why We Request Google User Data

Salesforce Form Today requests access to Google Forms data only to enable the core functionality of synchronizing form responses into Salesforce.

Google user data is accessed strictly for the following purposes:

  • Listing Google Forms owned by the authenticated user
  • Reading form questions and structure
  • Retrieving form responses submitted to selected forms

Access to Google data occurs only after explicit user consent and only during user-initiated actions.

5. Data Usage Transparency

All Google user data accessed by the application is handled with transparency and minimal scope.

  • Google Form data is used only to create or update Salesforce records
  • Data is stored only inside the customer’s Salesforce organization
  • No Google data is stored on external servers or third-party databases
  • No data is used for advertising, analytics, profiling, or tracking
  • Data is never sold or shared with third parties

6. What This App Does Not Do

Salesforce Form Today:

  • Does not read Gmail data
  • Does not access Google Contacts
  • Does not access Google Calendar
  • Does not send emails on behalf of the user
  • Does not show advertisements
  • Does not track users
  • Does not perform background or silent data collection

7. User Control and Security

  • Users authenticate using Google OAuth 2.0
  • Users can revoke Google access at any time from their Google Account settings
  • Synchronization runs only when manually triggered by the user
  • Salesforce permission sets and profiles control data visibility
  • OAuth tokens are securely stored using Salesforce security mechanisms

8. User-Initiated and Controlled Access

Salesforce Form Today does not perform automated background data collection.

  • Synchronization runs only when manually triggered by the user
  • Users can stop synchronization at any time
  • Users can revoke Google access at any time from Google Account security settings
  • Access tokens are managed securely using Salesforce’s Named Credentials and OAuth mechanisms

9. Intended Users

The application is intended for:

Salesforce Administrators
Salesforce Business Users
Organizations
Using Google Forms for structured data collection
The app is designed for internal business operations and organizational workflows rather than public or consumer-facing data collection.
